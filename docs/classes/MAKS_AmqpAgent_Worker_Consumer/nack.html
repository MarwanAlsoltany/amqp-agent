MAKS\AmqpAgent\Worker\Consumer::nack

Unacknowledges an AMQP message object.

Starting from v1.1.1, you can use php-amqplib AMQPMessage::nack() method instead.

    [optional] The channel that should be used. The method will try using the channel attached with the message if no channel was specified, although there is no guarantee this will work as this depends on the way the message was fetched.
